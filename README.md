# Data-Lakehouse-with-Snowflake
A data lakehouse using Azure Blob Storage and Snowflake to ingest, clean, transform, and analyse a large YouTube trending video dataset by creating external tables, loading CSV and JSON data, joining trending video records, removing duplicates, fixing null values, and writing SQL queries to answer business queries on video trends across countries.

It focuses on analyzing a large dataset of trending YouTube videos from different regions over a long time period – August 2020 to April 2024. Implementing Microsoft Azure and Snowflake, the goal of the assignment is to demonstrate how to manage and analyze a large, raw, and semi-structured dataset in a Data Lakehouse architecture, including data ingestion and transformation, followed by data cleaning to ensure the dataset is consistent by identifying and resolving issues such as duplicates, missing values, and erroneous data entries. Subsequently, the aim is to derive meaningful insights and address certain queries concerning YouTube trends across different regions. Based on the analysis, identifying the optimal content strategy for launching a new YouTube channel is the ultimate objective.

### Overview of Key Stages
- Uploading the folder of raw data for trending YouTube videos and metadata of the categories onto the Azure cloud storage.
- Data ingestion and converting it to Snowflake tables.
- Data cleaning to ensure accuracy and consistency.
- Data analysis to derive significant insights and respond to particular business inquiries concerning regional trends in YouTube videos.
